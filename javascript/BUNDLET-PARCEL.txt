- parceljs.org
-- Get Started -> Building a web app with parcel -> follow instructions to install it
--- create a new Git repository, install dependencies, and setup a few source files you can modify.
  npm create parcel vanilla my-parcel-app
  cd my-parcel-app
  npm start

--- in the project directory execute:
      npm install --save-dev parcel -> this did not work on my notebook until i shutdown VPN
--- this will add node_modules/ folder with many modules in your project
--- node_modules/* modules/libraries are not sent to client -> project size will not increase
--- open root_directory/package.json
---- "devDependencies" contains parcel -> means that it is used only in development. files will not be sent to client
----- --save-dev: Add the package to the devDependencies section of package.json.
----- --save-dev: Add the package to the dependencies section of package.json.
---- "dependencies" contains everything needed for the production (sent to client)

--- parcel has a development server built in, which will automatically rebuild your app as you make changes.
    To start it, run the parcel CLI pointing to your entry file:

---- npx parcel entry_file (such as src/index.html) -> this builds the final project (sent to client)
----- in unexplainable errors (normally after bug fixes) -> delete .parcel-cache directory from the root directory

- There is a better way than executing parcel by "npx parcel entry_file":
-- open package.json
  {
    "name": "starter",
    "version": "1.0.0",
    "description": "",
    "source": "index.html", // -> entry point -> see "scripts.start"
    "scripts": {
      "start": "parcel", // starts dev environment (e.g. starts server on localhost) -> on terminal: npm run start OR nom start
                         // assure that entry point it correctly configured -> see "source"
      "build": "parcel build" // makes final project for production -> on terminal: npm run build
    },
    "author": "",
    "license": "ISC",
    "devDependencies": {
      "eslint": "^8.35.0",
      "eslint-config-airbnb-base": "^15.0.0",
      "eslint-plugin-import": "^2.27.5",
      "parcel": "^2.9.3"
    },
    "dependencies": {
      "isomorphic-dompurify": "^1.7.0",
      "lodash": "^4.17.21"
    }
    "browserslist": "> 0.5%, last 2 versions, not dead",
  }

- npm run build:
-- before running command remove dist folder
-- create a folder called dist, which contains the final project with all files (.html/.css/.js/.map/images...)
-- each filename contains some hash information
-- there is a npm library called serve -> executing serve dist executes the final project

-- browserslist: use it to decide which browsers your app should support
--- not dead: exclude dead browsers such as IE
--- > 0.5%: Support browsers with more than 0.5% global market share
--- last 2 versions: This helps ensure users on recently updated browsers are covered.

Bundlers – Unique Identifiers

- npm i -g serve (or http-server): -> this did not work on my notebook until i shutdown VPN
-- i: shortcut for install
-- -g: global -> level of the entire device

- the final product contains hashes (unique ids) in filenames.
  each time the file's content changes -> after building the project,
  some files are cached & not reloaded every time.
  the filename will get a new hash. On loading website, the browser see that
  the filename has been changed and will invalidate cache & will reload the
  website
