Built in and Custom Data Attributes:
- Builtin Attributes
-- element.attribute (getter & setter) (in the html element, and not in the CSS file. CSS hat a higher priority)
--- document.querySelector("img").width -> 100
--- document.querySelector("img").getAttribute("width") -> 100 (same as .width)
--- document.querySelector("img").width = 200
--- document.querySelector("img").setAttribute("width", 200) (same as .width = 200)
-- console.dir(elementName): shows all properties of the element, such as width/height...

- Custom Attributes:
-- It is valid in HTML to add custom attributes in html elements, e.g. <img name="myimg">
-- Problem is that after a JS update the element might get a new standard attribute with the name
-- Solution: Custom-Attributes should start with the prefix "data-" -> <img data-name="myimg">
-- element.dataset: has all custom attributes as a DOMStringMap
--- e.g. img.dataset -> DOMStringMap {name: 'myimg'} (without the prefix "data-")

- Creating HTML
    const li = document.createElement('li')
    li.classList.add('skills__column')
    li.dataset.name = "typescript"

    const h2 = document.createElement('h2')
    h2.classList.add('skills__title')
    h2.innerText = "Typescript"

    const span = document.createElement('span')
    h2.classList.add('skills__years')
    h2.innerText = "2 years"

    const skillsList = document.querySelector('.skills__list');
    # next 3 lines will touch DOM 3 times -> performance issue because DOM will recalculate everything after each write access
    skillsList?.appendChild(li);
    li.appendChild(h2)
    li.appendChild(span)

    # better solution
    li.appendChild(h2)
    li.appendChild(span)
    skillsList?.appendChild(li);

    # DRY: Don't repeat yourself
    const createNewElement = (tag, className, text, dataName) => {
      const newElement = document.createElement(tag);
      newElement.classList.add(className);
      if(text) { newElement.innerText = text; }
      if(dataName) { newElement.dataset.name = dataName; }
      return newElement;
    }

    const li = createNewElement('li', 'skills__column', undefined, 'typescript');
    const h2 = createNewElement('h2', 'skills__title', 'Typescript'); # last argument is undefined by default
    const span = createNewElement('span', 'skills__years', '2 years');

    li.appendChild(h2)
    li.appendChild(span)
    # skillsList?.appendChild(li);
    # insert after element javascript
    const jsElement = document.querySelector('[data-name="Javascript"]); # data-name is case sensitive
    skillsList?.insertAdjacentElement('afterend', li);

--- one disadvantage is that string html code are not expandable, e.g. we cannot add a new attribute
--- solution:

    const liElement2 = document.createRange().createContextualFragment(liElement); # converts string to actual elements
    # typeof liElement2 -> document-fragment
    liElement2.querySelector('h2');
