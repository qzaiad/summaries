`npm init` is a command that creates a **`package.json`** file for a Node.js project.

The `package.json` file stores metadata about your project, such as:

* Project name
* Version
* Description
* Entry point
* Scripts
* Dependencies
* License

---

## Basic Example

Suppose you create a new project:

```bash
mkdir my-app
cd my-app
npm init
```

npm asks a series of questions:

```text
package name: (my-app)
version: (1.0.0)
description:
entry point: (index.js)
test command:
git repository:
keywords:
author:
license: (ISC)
```

After answering, npm generates:

```json
{
  "name": "my-app",
  "version": "1.0.0",
  "description": "",
  "main": "index.js",
  "scripts": {
    "test": "echo \"Error: no test specified\" && exit 1"
  },
  "author": "",
  "license": "ISC"
}
```

---

## `npm init -y`

Most developers use:

```bash
npm init -y
```

The `-y` means:

```text
yes to all defaults
```

No questions are asked.

It immediately creates:

```json
{
  "name": "my-app",
  "version": "1.0.0",
  "main": "index.js",
  "scripts": {
    "test": "echo \"Error: no test specified\" && exit 1"
  },
  "license": "ISC"
}
```

---

## Why is `package.json` important?

When you install packages:

```bash
npm install express
```

npm updates `package.json`:

```json
{
  "dependencies": {
    "express": "^5.1.0"
  }
}
```

This allows others to install the same dependencies with:

```bash
npm install
```

## Common Workflow

### 1. Create project

```bash
mkdir api-server
cd api-server
```

### 2. Initialize npm

```bash
npm init -y
```

### 3. Install packages

```bash
npm install express
```

Result:

```json
{
  "name": "api-server",
  "version": "1.0.0",
  "dependencies": {
    "express": "^5.1.0"
  }
}
```

### 4. Add scripts

Edit `package.json`:

```json
{
  "scripts": {
    "start": "node index.js"
  }
}
```

Run:

```bash
npm start
```

npm executes:

```bash
node index.js
```

## `npm init <initializer>`

`npm init` can also run project generators.

Examples:

```bash
npm init react-app my-app
```

or

```bash
npm init vite@latest
```

These are shortcuts for creating projects from templates.

For example:

```bash
npm init vite@latest my-vue-app
```

creates a complete Vite project structure instead of just a `package.json`.

## Summary

| Command                | Purpose                                  |
| ---------------------- | ---------------------------------------- |
| `npm init`             | Interactive creation of `package.json`   |
| `npm init -y`          | Create `package.json` using defaults     |
| `npm install <pkg>`    | Add dependency and update `package.json` |
| `npm init vite@latest` | Run a project scaffolding tool           |

Think of **`npm init`** as the command that turns a regular folder into an **npm-managed project** by creating the `package.json` file that npm uses to track configuration and dependencies.
