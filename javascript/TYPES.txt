Primitive Types:
- String:
-- let s = "Qassem"; // or 'Qassem'
- Number:
-- let n = 100; // or 100.0
- Boolean:
-- let b = true/false;
- Null:
-- let nu = null;
- Undefined:
-- let un; // not assigned any value
- Symbols:
-- let sy = Symbol(); // gives a unique identifier

Non-Primitive Type:
- Object: Collection of types
--  let ob = {
        "key": "xyz",
        "name": "Qassem",
        "age": 44
    }

--  let ar = ["Qassem", 44] // equivalent to let ar = {"0": Qassem, "1": 44};

- Javascript has dynamic typing, i.e. variables may change also type
- typeof null -> object

Null and Undefined
- Undefined: variable is declared but not assigned a value, or user did not write anything
- Null: has the value nothing

    function calculateTakenCourses(courses){
        let result;
        if(courses === undefined){
            result = "User did not answer the question";
        }
        else if(courses === null){
            result = "User did not take any course";
        }
        else if(courses === "number"){
            result = `User took ${courses} courses)`;
        }
        else if(courses === "object"){
            result = courses.join(", ");
        }
        return result;
    }


Objects: Equivalent to C-Structs
- Order is not important
    let ob = {
        key1: value1,

        key3: value3
    };

- if order is important -> use arrays
    let arr = [value1, value2, value3, ...]

- keys must be unique


Naming Convention

- UX: User Experience
-- Rules that make the user familiar witht he website.
--- Everything is in his right place such as buttons

- DX: Developer Experience
-- Rules that make your code familiar, so that others can use the code after you
-- www.syncfusion.com -> 10 naming convensions
-- Variable names: camelCase
-- booleans: isGood, areIdeal, hasOwner
-- functions: getName, setName
-- constants: LEG, TAIL...
-- Classes PascalCase: class DogCartoon...
-- Components PascalCase: function DogCartoon(roles) { return (<div>...</div>); }
