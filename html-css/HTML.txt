- <h1> element should only have only one primary heading h1:
 - The web browser can handle multiple h1 elements
 - Good Practice to have only one h1 element
 - It is a rule for Search Engine Optimization (SEO) which helps
   showing you website in google results and get a good result (tarteeb)
- <b> element is not a good practice and should not be used in html5 (replaced by <strong>)
  because html5 cares more about Semantic HTML
- Semantic HTML:
 - Writing a code which has a meaning (or DALAALAH)
 - Using semantic html gives your website a higher ranking for Search Engine Optimization (SEO)
- <i> element: italic -> replaced by <em> element

- <strong> should replace <b>
- <em> (emphasize) should replace <i>
- <img src="" alt="" width="" height="" /> self closing element/tag>
 - attribute alt := alternative text
  1) Allows search engines to know the content of the image
     Helps to better website index the website and give it a better ranking
  2) SCREEN READER use it to describe the img content for blind people
  3) If for some reason the image could not be loaded -> the alt text will be displayed

- Page Structure: USEFUL FOR PEOPLE WITH
 - header: <header></header> usually contains main title & search/browser tools/links
  - page navigation: <nav></nav> usually contains links in the same page
   - it logically group elements
    - all elements in a group can be treated e.g. in CSS as a group
 - footer: <footer></footer> usually contains additional info as links
 - We might have multiple <nav> elements in one section. To make is
   helpfull for people with accessibilities define a text which should be read
   by the Screen Reader by the attribute aria-label. For Example:
   <nav aria-label="primary navigation">
   <a>Courses...</a>
   </nav>
   <nav aria-label="language navigation">
   <a>Arabic</a>
   <a>English</a>
   </nav>

 - article: <article></article> inside body. Sematically describes content as an article
  - An article might again contain a header element

Multilingual Websites: There are two ways to do it:
- By using JavaScript (used in Single Page Applications SPA)
- For each language create an HTML file with a meaningfull name such as index-ar.html
 - By replacing the english content with arabic content the page will show the arabic text
   for left to right (ltr) instead of right to left (rtl) -> user attribute html.dir
  - <html lang="ar" dir="rtl/ltr/auto"
  - attribute dir can be given to any element in the body
   - e.g. in index-ar.html we write a paragraph in english: <p dir="ltr">Hello World!</p>

- <aside>:
يُستخدم لعرض محتوى جانبي أو ثانوي مرتبط بالمحتوى الرئيسي، مثل الملاحظات، الروابط الإضافية، الإعلانات، أو مربعات المعلومات
بمعنى آخر، هو مساحة لعرض معلومات مساعدة أو مكملة وليست جزءًا أساسيًا من النص الرئيسي.

    </aside>

بمعنى آخر: هو عنصر يُنظّم المحتوى في وحدات منطقية داخل الصفحة.
    <section>

- <main>
يُستخدم لاحتواء المحتوى الرئيسي للصفحة — أي الجزء الأهم الذي يعبّر عن هدف الصفحة، ويكون فريدًا ولا يتكرر في كل صفحة مثل الترويسة أو التذييل.
باختصار: <main> يجمع أهم محتوى يريد المستخدم رؤيته.
    <main>
    <h1>مرحبًا بكم في موقعي</h1>
    <p>هذا هو المحتوى الرئيسي الذي يقدّم فكرة الموقع.</p>
    </main>


HTML Tables:
- has also been used earlier in page layout for dividing the page into
  header, body, footer and side -> NOT RECOMMENDED NOW (OLD FASHION)
- <table border="1" cellspacing="10" cellpadding="10" summary="table summary"></table>
 - attrbiute border is deprecated -> use CSS instead
 - attribute cellspacing: distance between adjacent cells
 - attribute cellpadding: internal distance between words and border
 - attribute summary: for SEO and better accessibility (used by SCREEN READER)
- <caption>Table Title</caption>
- <tr></tr>: table row
- <th id="id1"></th>: table head
 - attribute id: used to connect <td>s (cells) with headers (by using <td headers="th.id">) (SEO)
- <td headers="id1" width="100%" valign="top"></td>: table data
 - attribute headers: connects cell to header (SEO)
 - attribute width:
 - attribute valign: align content vertically top
 - attribute rowspan:
 - attribute colspan:
- Semantic HTML: For optimizing SEO and better ranking
 - <thead></thead>
 - <tbody></tbody>
 - <tfoot></tfoot>
 - The browser add automatically sematic elements if missing or not given.
  - if there are semantic headers -> browser adds <tbody>

- Example for 2x2 table:
    <table border="1">
    <caption>Table Title</caption>
      <thead>
        <tr>
          <th id="header1">Head1</th>
          <th id="header2">Head2</th>
        </tr>
      </thead>
      <tbody>
        <tr>
          <td headers="header1">Data1</td>
          <td headers="header2">Data2</td>
        </tr>
        <tr>
          <td colspan="2"></td> <!-- cell spans for two columns -> merging 2 horizontal cells -->
        </tr>
        <tr>
          <td rowspan="2"></td> <!-- cell spans for two rows -> merging 2 vertical cells -->
        </tr>
      </tbody>
      <tfoot>
        <tr>
          <td colspan="2">This is a footer</td>
        </tr>
      </tfoot>
    </table>

 - Attributes:
  - required
  - min/max/step: min=18, max=100, step=2 -> 19 is not accepted -> accept onyl 18, 20, 22,...,100
  - minLength/maxLength
  - pattern="\d{5}": Regular Expression

<form action="url" method="get (default)/post"> # url which should process the form data
 # method=get has a limit -> max url length is 2048
 # method=post: data sent in HTTP-Request and is encrypted if HTTPs and has no length limit
 <fieldset> # groups all elements logically (surrounded by a line)
  <legend>Your Information</legend>
  <label for="name" minlength="5">Name:</label> # clicking on the text "Name:" focus on the corresponding input
  <input id="name" type="text" [required] />
  <label for="email">Email:</label>
  <input id="email" type="email" placeholder="example@gmail.com" />
  <label for="reason">Reason for contact:</label>

    <option value="">Select a reason</option>           # no value

    <option value="admission">Admission Information</option>
  </select>

  <textarea id="message" rows="4" cols="30">Default Text</textarea>
